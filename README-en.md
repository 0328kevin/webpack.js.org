## 内容进度

现在我们已经覆盖到大部分积压的_缺失文档_，
我们开始大量 review 网站内容的每个部分，
以便对其进行适当的排序和结构化。
下面的 issues 可以提供一个不错的大体概念，包括放置在哪里和预计放置位置：

- [指南 - 简化][1]
- [概念 - Review and 组织][2]
- [API - v4 重写][8]

我们还没有为其他部分创建 issues，但很快进行。
有关开发相关的工作，请查看[常规 - 更新和修复][3]。
